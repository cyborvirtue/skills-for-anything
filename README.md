# Skills for Anything

> A curated collection of open-source AI skills, skill packs, and skills resources across coding, research, writing, design, data, product, marketing, and more.

Skills for Anything is a GitHub-first guide to reusable AI capabilities.

Most collections are organized by platform. This one is organized by what you want to do.

This list is selective, not exhaustive. The goal is to surface high-signal projects and standout skill directories without turning the repository into a link dump.

Last reviewed: March 10, 2026

## Why This Exists

Open-source skills are spreading across many ecosystems, but discovery is still fragmented.

Some projects are vendor-specific. Some are marketplaces. Some are just one excellent skill hidden inside a much larger repository. Skills for Anything exists to make that landscape easier to browse by domain instead of by platform.

## What Counts as a "Skill" Here?

For this list, a skill is one of the following:

- a reusable AI capability bundle
- a structured skill pack or plugin marketplace
- a skills installer, registry, or portability layer
- a curated skills directory
- a high-signal skill directory inside a larger repository

Most entries point to GitHub repositories. Some point directly to a specific skill directory when that is the most useful unit for discovery.

## How to Use This List

- Start with [Featured Foundations](#featured-foundations) if you want the quickest orientation.
- Jump to a domain if you already know the type of work you want to improve.
- Use [Standards, Marketplaces & Discovery](#standards-marketplaces--discovery) if you want installers, registries, or ecosystem entry points.
- Expect uneven category sizes. Some domains are mature today; others are still emerging.

## Submission Criteria

An entry should pass at least three of these checks:

- clearly represents a reusable skill, skill pack, skills directory, or skills infrastructure layer
- has obvious domain relevance
- has visible utility beyond a one-off demo
- shows traction, active maintenance, or ecosystem importance
- makes the list more useful for discovery

This repository does not try to collect every repo that says "skill" once in a README.
